## ADDED Requirements

### Requirement: App supports injected and MetaMask wallet connection
The app SHALL allow users to connect and disconnect using injected browser wallets and MetaMask only.

#### Scenario: Wallet options shown
- **WHEN** no wallet is connected
- **THEN** the wallet UI offers injected and MetaMask connection options without WalletConnect

#### Scenario: Connected account shown
- **WHEN** a wallet connection succeeds
- **THEN** the wallet UI shows the connected account address in shortened form and offers a disconnect action

#### Scenario: WalletConnect omitted
- **WHEN** the wallet UI renders available connectors
- **THEN** WalletConnect is not shown as an option

### Requirement: App supports 0G Galileo testnet chain readiness
The app SHALL configure and validate the 0G Galileo testnet for wallet-connected usage.

#### Scenario: Correct chain connected
- **WHEN** the connected wallet is on the configured 0G Galileo testnet chain
- **THEN** the chain readiness UI shows a connected 0G Galileo state

#### Scenario: Wrong chain connected
- **WHEN** the connected wallet is on a chain other than the configured 0G Galileo testnet chain
- **THEN** the chain readiness UI shows a wrong-network state and offers a switch or add-chain action

#### Scenario: Chain ID verified before implementation
- **WHEN** implementation finalizes the 0G Galileo chain configuration
- **THEN** the configured chain ID matches the value returned by `eth_chainId` from `https://evmrpc-testnet.0g.ai`

### Requirement: App initializes 0G Direct Compute from wallet signer
The app SHALL use the connected wallet signer to initialize 0G Direct Compute for inference.

#### Scenario: Broker unavailable before wallet connection
- **WHEN** no wallet is connected
- **THEN** the Direct Compute broker is not initialized and the UI explains that wallet connection is required

#### Scenario: Broker initializes on correct chain
- **WHEN** a wallet is connected on 0G Galileo testnet
- **THEN** the app attempts to initialize a 0G Compute Network Broker using a wallet-backed ethers signer

#### Scenario: Broker initialization failure
- **WHEN** broker initialization fails
- **THEN** the UI shows an inline Direct SDK error and keeps the chatbox available in a disabled or blocked state

### Requirement: App discovers inference providers and metadata
The app SHALL discover available 0G inference providers and expose selected provider/model readiness.

#### Scenario: Provider list loading
- **WHEN** the Direct Compute broker is initialized
- **THEN** the app requests available inference services and shows a loading state until the request completes

#### Scenario: Provider available
- **WHEN** at least one chatbot inference service is available
- **THEN** the app shows the selected provider and model metadata in the compute readiness UI

#### Scenario: No provider available
- **WHEN** no chatbot inference service is available
- **THEN** the app shows a no-provider state and prevents prompt submission

### Requirement: App sends wallet-signed 0G inference prompts
The app SHALL send chat prompts through the selected 0G Direct Compute provider using request headers generated by the wallet-backed broker.

#### Scenario: Prompt blocked before readiness
- **WHEN** wallet, chain, provider, or funding readiness is incomplete
- **THEN** the app prevents sending a 0G inference prompt and shows the blocking readiness reason

#### Scenario: Prompt sent when ready
- **WHEN** wallet, chain, provider, and Direct SDK request headers are ready
- **THEN** submitting a chat prompt sends a chat completions request to the selected provider endpoint with the selected model and generated request headers

#### Scenario: Response displayed
- **WHEN** the selected provider returns a chat completion response
- **THEN** the assistant response is appended to the chat conversation

#### Scenario: Optional response verification
- **WHEN** response verification data is available from the provider response
- **THEN** the app can process the response through the Direct SDK verification flow and surface verification status
